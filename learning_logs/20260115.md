## 갖다 놓는(배치하는) `HTML` / 꾸며주는 `CSS` / 동작 시키는 `JS`
- Java를 배우면서 작성한 것은 code라는 표현을 썻지만 , HTML + CSS로 작업한것은 문서라고함. 즉, 얘네 둘을 이용해서 만든것은 프로그램이 아니라 document에 해당.

- 웹 페이지에 들어갈 요소들을 배치하고 디자인하는 정도는 .html 파일과 .css파일은 브라우저 라는 프로그램(크롬) 에서 불러와서 보여주는 문서을 의미.

- 하지만 JS의 경우 서버로부터 전달 받아 브라우저 에서 동작하는 파일에 해당. 하지만 Node.js등 컴퓨터에 설치된 Js엔진에서 python / java와 같은 언어처럼 복잡한 알고리즘을 정의하고 컴퓨터에 명령을 내릴수있음.

이상을 이유로 JS는 프로그래임 언어이기 때문에 code라고 하고, HTML + CSS는 문서라고 함.

## 웹 디자니어 vs 웹 퍼블리셔
웹 디자이너가 작업한 결과물을 HTML + CSS문서로 작성하는 사람을 우리나라에선 웹 버플리셔라는 표현을 함. 여기에 JS 까지 적용하여 기능을 만들어 낼수있는 사람을 웹 개발자라고 할수있음.

```HTML
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>여기가 사실상 제목</title>
</head>
<body bgcolor="5f5f5f">
    <h1>Hello, World!</h1>
    .
</body>
</html>
```
`! + Tab` 으로 자동 생성된 것은 이상과 같은. 이는 웹 개발 시에 기계 또는 시스템에 전달하는 각종 정보로, _HTML 문서작성시에 꼭 필요한 내용_ 들을 담고 있음. 문서 제목, 작성자, 사용언어 및 각종 메타정보(메타 데이터) 등으로 구성 되어있는데, 어짜피 형식 자체는 동일 하기때문에 일일이 타이핑 할 필요 없이 단축키를 지원. 이를 emmet이라고 하는데 `!+Tab`도 이중 하나에 속함. 앞으로 에밋 이라고 했을때, 특정 문서 작성을 빠르게 해주는 단축키로 생각하면 됨. 그리고 `!+Tab`은 .html 파일 생성할때마다 맨 처음에 하는 작업이 됨.

 `<html lang="en">`의 의미
- lang 속성은 HTML 문서의 웹 사이트가 어떤 언어로 되어있는지 _컴퓨터에게 알려주는_ 역할을 함. 이걸 지정해줬을때 구글 / 네이버 등에서 검색할때 한글로 된 웹 사이트를 찾는것이 가능. 또한 시각장애인들을 위한 컴퓨터가 웹 사이트를 분석할때도 쓰임. 그리고 맥 사용자의 경우 lang="en"인데 캡스락 눌러서 한글로 웹 페이지 작성하면 기본 서체가 궁서체가 됨.

## 택스트 관련 태그
웹 페이지에서 텍스트를 지정할때는 내용에 따라 글자를 강조 하거나 중요하다고 표시하게 됨. 나무위키 같은 곳을 보면 괜한 뻘글쪽에 취소선이 그려져있는 형식으로 특정 텍스트의 스타일을 다양하게 조절하는 태그들도 존재함.
1. `<b>` : bold를 의미
2. `<strong>` : `굵게`를 의미
3. `<i>` : 이텔릭체를 의미
4. `<em>` : `기울임체`를 의미

ch01_basic_tags에 `strongtag.html파일`을 만들고, `lang="ko"`로 바꿔서 body의 bgcolor를 `5f5f5f`로 만든 다음에 라이브 프리뷰를 켜서 세팅.

- 옛날에는 HTML과 CSS가 분리되지 않았기 때문에 웹 페이지 상에서의 서체를 굵게 지정하거나 기울이는 것을 전부 다 태그로 처리했었음.

- 5 버전 이후부터는 태그는 코드적으로 어떤 위상에 속하는지만 표시하는 지표로 쓰이고, 데코레이션은 CSS로 넘어가게 됨.

- 즉, b 태그와 i 태그는 시각적으로 강조하는 효과에 해당하고, strong 태그와 em 태그의 경우에는 해당 text들의 중요 토픽에 해당하는 부분이라는 것을 컴퓨터에게 알려주는 역할.

- 그러면 검색 엔진에서 특정 검색어를 지정 한다고 할때, 엔진은 strong이나 em 태그에 검색어와 동일한 부분이 잇는지를 체크하게 됨.

### 첨자 태그
위 첨자 및 아래 첨자라는 개념을 학습한 적이 있음. 보통은 영어시간이나 수학시간에 배움.
- 예시
1<sup>st</sup>, 2<sup>nd</sup>,
3<sup>rd</sup>, 4<sup>th</sup>

- 아래 첨자 예시
H<sub>2</sub>O

### 밑줄 태그와 취소선 태그
_이탤릭체_ 쓰거나 **이렇게** 강조하거나 하는 방식이 markdown에서 씀.

밑줄예시
<u>이렇게 쓰면 됨.</u>

취소선 예시
<s>이렇게 쓰면 됨.</s>


마찬가지로 예전엔 html상에서 다 처리했지만 이제는 css로 함.
